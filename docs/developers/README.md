# Раздел для разработчиков

Этот раздел нужен, когда уже мало просто пользоваться SpsFW как библиотекой и надо понимать его внутренние гарантии, ограничения и runtime-поведение.

Здесь нет повторения базовых пользовательских инструкций. Для этого уже есть основная документация в `../README.md` и тематические главы. Этот раздел дополняет их там, где нужно смотреть под капот.

## Что здесь искать

- [Очереди и transactional outbox](queue-outbox.md) — как устроены публикация сообщений, `LISTEN/NOTIFY`, Redis-замена для MySQL/MariaDB, lease-модель, идемпотентность и алгоритмические оценки.

## Когда сюда идти

- нужно понять, почему сообщение не публикуется сразу, а сначала попадает в БД;
- нужно выбрать wakeup-стратегию для PostgreSQL или MySQL/MariaDB;
- нужно оценить, где у очередей горячий путь, а где только фоновый housekeeping;
- нужно объяснить команде, что SpsFW гарантирует, а что гарантировать не может.
